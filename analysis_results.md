# EXIT 67 게임 분석 보고서

본 보고서는 일본의 인디 공포 게임인 ‘8번출구(Exit 8)’를 모티브로 하여 웹 기술만으로 구현된 **EXIT 67** 3D 미스터리 탈출 게임의 아키텍처와 핵심 로직, 구현 기술 및 67가지 이상현상에 대해 분석한 결과를 정리한 문서입니다.

---

## 1. 게임 개요 및 핵심 플레이 룰

*   **게임명**: EXIT 67
*   **장르**: 3D Horror Arcade Web Game (3D 공포 루프 탈출)
*   **핵심 플레이 룰**:
    *   플레이어는 무한히 반복되는 지하 복도 공간에서 탈출해야 합니다.
    *   **이변(이상현상)이 없을 경우**: 앞(Front Door)으로 전진하여 탈출을 진행합니다.
    *   **이변(이상현상)이 있을 경우**: 즉시 뒤(Back Door)로 돌아가서 복도를 빠져나와야 합니다.
    *   올바른 선택을 할 때마다 현재 출구 번호(`exitNumber`)가 **1씩 증가**합니다.
    *   잘못된 선택을 하거나 잘못된 문으로 나갈 경우 진행 상황이 **1 단계 후퇴**하며, 0 이하로는 떨어지지 않습니다.
    *   특정 위험한 이변(예: 새파란 손자국, 그림자 추격자 등)에 닿거나 대처하지 못하면 **게임오버**가 되며, 진행도가 완전히 초기화되어 **0번 복도로 리셋**됩니다.
    *   출구 번호가 **67**에 도달하면 전방의 문이 사라지고 지상으로 올라가는 **3D 계단**이 형성되며, 계단을 끝까지 올라가면 탈출에 성공(Game Clear)하게 됩니다.
    *   **이변 도감(Anomaly Dictionary)** 기능이 내장되어 있어, 원하는 이변을 선택하면 해당 이변이 즉시 활성화된 복도로 순간 이동하여 바로 체험할 수 있습니다. (메인 메뉴의 '이변 도감 보기' 버튼을 통해 제공)

## 2. 기술 스택 및 아키텍처

EXIT 67은 별도의 상용 게임 엔진(Unity, Unreal 등)을 사용하지 않고, 오직 웹 표준 기술과 오픈소스 라이브러리를 활용해 구현되었습니다. 특히 **어떠한 외부 미디어 에셋(이미지, 오디오 파일 등)도 로드하지 않고**, 모든 리소스를 JavaScript 코드를 통해 실시간으로 절차적 생성(Procedural Generation)합니다.

*   **Core**: HTML5, Vanilla JavaScript
*   **Styling**: Vanilla CSS (`style.css`)
*   **3D Graphics**: Three.js (r128) - WebGL 기반 3D 렌더링
*   **Audio**: Web Audio API (합성 기반 절차적 사운드 엔진)
*   **Build Tool**: Vite (개발 및 서빙 환경 제공)

### 주요 프론트엔드/게임 메커니즘
1.  **Pointer Lock API**: 마우스 클릭 시 화면 중앙으로 포인터를 가두고, 마우스 이동 입력을 감지해 자연스러운 FPS(1인칭) 시점 회전(Yaw, Pitch)을 구현합니다.
2.  **Raycasting**: 화면 중앙을 기준으로 광선(`THREE.Raycaster`)을 쏴 플레이어가 문(Front, Back, 특정 이변 문)을 클릭하여 상호작용하는 물리 판정을 처리합니다.
3.  **Collision Clamping**: 플레이어의 이동 범위를 복도의 벽 안쪽(X: -1.6 ~ 1.6)과 출구 문 영역(Z: 복도 길이에 따른 제한) 내로 제한하여 3D 맵 탈출을 방지합니다.
4.  **Fog Effect**: `THREE.FogExp2`를 사용하여 복도 끝이 어둡게 흐려지도록 포그 효과를 적용함으로써 지하도의 답답하고 공포스러운 심리적 공간 연출을 강화하였습니다.

## 3. 절차적 사운드 엔진 (ProceduralSoundEngine)

`sound.js`에 구현된 `ProceduralSoundEngine`은 외부 사운드 파일을 로드하지 않고 **Web Audio API**의 오실레이터(Oscillator), 노이즈 버퍼(Noise Buffer), 바이쿼드 필터(Biquad Filter), 스테레오 패너(Stereo Panner) 등을 조합하여 실시간으로 효과음과 환경음을 합성합니다.

| 음향 효과 | 합성 기술 방식 | 인게임 내 활용 |
| :--- | :--- | :--- |
| **Ambient Hum** | Sawtooth 및 Sine 오실레이터를 미세하게 튜닝하고 0.1Hz LFO로 필터 주파수를 변조하여 낮게 웅웅거리는 드론(Drone) 베이스 생성 | 지하 복도의 불길한 환경음 및 백그라운드 노이즈 |
| **Footsteps** | 힐 탭 노이즈(Triangle) + 목재 반향(Sine) + 고주파 스커프 노이즈(Noise)를 조합해 물리적 보행 속도에 따라 재생 | 플레이어 및 추격자의 이동(또각또각) 발소리 |
| **Flashlight Click**| 고주파 Triangle 파형을 짧게 끄며 Highpass 필터링 | 손전등(X키) 스위치 켜고 끄는 소리 |
| **Door Bang** | 낮은 주파수의 우드 타격음(Triangle)과 노이즈 크래시를 3회 급격히 바운스하여 연속 타격 처리 | 11번 이변(두드리는 문)의 거친 타격 소리 |
| **Scream & Horror** | 극도로 튜닝된 25Hz LFO 진동을 두 개의 톱니파(Sawtooth)와 사각파(Square)에 걸고 밴드패스 필터를 exponential 스윕 | 29번 이변(비명 지르는 졸라맨) 등의 점프스케어 비명 |
| **Rain & Thunder** | 화이트 노이즈를 밴드패스 필터링하여 지속 루프 생성 + Triangle 파형 및 저주파 로우패스 노이즈 감쇄로 벼락 시뮬레이션 | 56번 이변(빗소리와 천둥)의 기후 연출 |
| **Clock Tick** | Triangle 파형의 고주파/중주파 피치를 번갈아 재생해 똑딱거리는 기계음 합성 | 54번 이변(괘종시계 소리)의 압박감 조성 |
| **Whispering** | 스피치 대역(1.2k~2.0kHz)의 노이즈 버스트를 필터링하고 음절 단위로 볼륨을 펄스 진동시켜 기괴한 목소리 연출 | 55번 이변(속삭이는 벽)의 청각적 공포 연출 |

## 4. 절차적 텍스트 및 머티리얼 시스템 (AnomalySystem)

`anomalies.js`에 구현된 `AnomalySystem`은 HTML5 Canvas API를 이용하여 동적으로 2D 이미지를 즉석에서 그려내고, 이를 `THREE.CanvasTexture`로 변환하여 Three.js 메쉬에 매핑하는 방식을 채택합니다.

*   **나무 마룻바닥 (Wood Floor)**:
    *   시드 기반 난수(Seeded Random)를 활용해 타일링 경계가 맞아떨어지는 나뭇결(Wood Grain) 곡선, 옹이(Knot) 무늬, 베벨 하이라이트 및 음영을 실시간 2D 드로잉하여 실감 나는 나무 복도 바닥 텍스처를 렌더링합니다.
*   **미술/전시 포스터 (Art Gallery Poster)**:
    *   색상 반전, 외계 문자 표시, 내부의 미니 복도 도식('YOU ARE HERE') 그리기 등의 연출을 캔버스를 통해 동적으로 제어합니다.
*   **졸라맨 포스터 (Stickman Poster)**:
    *   정상적인 졸라맨 이미지 외에도, 피투성이 손자국(Bloody Handprint), 비명 지르는 얼굴 상태, 백지(Blank) 포스터 등을 실시간 좌표 연출로 변경해 머티리얼을 업데이트합니다.
*   **출구 표지판 (Exit Sign)**:
    *   LED 백라이트 글로우 효과(ShadowBlur 적용), 음수 간판(-5), 데스 넘버(444), 글리치 문자($&%), 출구 없음 텍스트 등을 동적으로 캔버스에 렌더링하여 안내 표지판 오브젝트에 부착합니다.

## 5. 67가지 이상현상 (Anomalies) 상세 일람

게임에 구현된 총 67가지의 독창적인 이상현상 목록과 각 현상의 구체적인 작동 방식 및 연출 효과를 정리한 테이블입니다.

> [!IMPORTANT]
> 1~67번의 모든 이상현상은 기본 복도 구조에서 미세하거나 극적인 변화를 일으키며, 플레이어가 이를 발견하면 즉시 뒤쪽 문으로 가야 세이브를 잃지 않고 통과할 수 있습니다.
